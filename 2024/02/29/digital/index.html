【Original】The Mysteries of Numbers: Exploring the Magical Numbers in the Field of Computing

Computer

Posted by Liyane on Saturday, March 2, 2024

Categories: [Technology/Tech]

Introduction

In the sea of digital stars, there exists a universe that operates in a counterintuitive way for most people. It’s not the mother of all things, but it is undoubtedly the father of modern technology. Yes, we’re talking about the role of numbers in the field of computing. Not so ordinary, yet extremely crucial, from the binary system that seems mundane to the deep and mysterious hexadecimal, the principles, applications, and stories behind it all are fascinating.

Imagine: What would our world be like without numbers? No precise calculations, no exquisite graphic designs, no thrilling video games, and not even the smartphones we use daily for communication. Indeed, numbers are the foundation of modern electronic technology, especially in the realm of computing.

Binary

Everything in the computing world begins with binary, a world made up of 0s and 1s. Each binary digit (bit) represents the smallest unit of data, and 8 bits make up a byte, which is the basic framework for computers to process and store data. A byte can represent 256 (2^8) different states, forming the most fundamental character set that computers can recognize.

A Bit: 0 and 1

In the world of computers, 0 and 1 are super heroes. These two numbers form the binary system, which is how computers understand the world. Interestingly, they actually represent the states of electrical current being on and off. Yes, just like the light switch at home, only computers constantly ’turn on’ and ’turn off’ them at an astonishing speed.

A Byte: 2^8 = 256

When 8 bits come together to form a byte, magic happens. You now have 256 possible combinations to represent a character, such as letters, numbers, or other symbols. From the ASCII table to your computer screen displaying ‘Hello, World!’, it’s all thanks to the power of a byte.

Two Bytes: 2^16 = 65,536

Two bytes can represent 65,536 possible combinations.

Computer System Port Numbers: The Limit of 0-65,535

In computer networking, a port is a logical construct used to differentiate between different processes or types of network data. Each IP address is assigned 65,536 possible port numbers, ranging from 0 to 65,535. In network communication, ports are allocated to specific services, such as HTTP typically using port 80 and HTTPS using port 443. This number not only reflects the use of binary but also demonstrates the logic of resource allocation in computer networks.

Four Bytes: 2^32 = 4,294,967,296

IPv4 Address

IPv4 address is the cornerstone of internet communication, consisting of 4 bytes, usually represented in the format X.X.X.X. With each byte capable of representing 256 states, the theoretical maximum number of IPv4 addresses is 256^4 = 4,294,967,296, exceeding 4 billion. This number seemed sufficient in the early days of the internet, but with the explosive growth of devices, the exhaustion of address space has become a problem, driving the development of IPv6.

32-bit Operating System Addressing Limit

In a 32-bit system, the CPU registers are 32 bits wide, which directly affects its memory addressing capability. Theoretically, a 32-bit system can address (2^32) different addresses, meaning they can directly access 4GB (Gigabytes) of memory space, or (4,294,967,296) bytes. However, in practice, the operating system usually reserves a portion of the address space for the kernel, resulting in actual usable memory being less than 4GB.

Integer Data Type

In many programming languages, an ‘integer’ data type is typically defined as four bytes. Its range is:

  • Signed: -2,147,483,648 to 2,147,483,647
  • Unsigned: 0 to 4,294,967,295

Eight Bytes: 2^64 = 18,446,744,073,709,551,616

64-bit Operating System

A 64-bit system’s CPU registers are 64 bits wide, significantly increasing its memory addressing capability. A 64-bit system can theoretically address (2^64) different addresses. This equals 16EB (Exabytes), or approximately (18.4) billion GB. In reality, this number far exceeds the memory requirements of any personal computer system as of now (as of 2023), so the actual addressing capability of modern 64-bit operating systems is limited by hardware and operating system design. For example, the home version of Windows 10 64-bit supports up to 128GB of RAM, while the professional version supports up to 2TB (Terabytes) of RAM.

1024: The Leap from Bytes to Kilo

Over time, people realized that using powers of 2 to define storage units could cause confusion when communicating with conventional decimal units (such as metric units). For instance, in marketing and product specifications, 1KB is represented as 1024 bytes, while in other contexts, people might be more accustomed to equating 1KB with 1000 bytes.

To resolve this ambiguity, the International Electrotechnical Commission (IEC) introduced a new set of prefixes for binary storage capacity units in 1998, such as “Kibi-”, “Mebi-”, “Gibi-”, etc. Thus, according to the IEC standard:

  • 1 KiB (Kibibyte) = (2^10) Bytes = 1024 Bytes
  • 1 MiB (Mebibyte) = (2^20) Bytes = 1024 KiB
  • 1 GiB (Gibibyte) = (2^30) Bytes = 1024 MiB
  • 1 TiB (Tebibyte) = (2^40) Bytes = 1024 GiB
  • 1 PiB (Pebibyte) = (2^50) Bytes = 1024 TiB
  • 1 EiB (Exbibyte) = (2^60) Bytes = 1024 PiB
  • 1 ZiB (Zebibyte) = (2^70) Bytes = 1024 EiB
  • 1 YiB (Yobibyte) = (2^80) Bytes = 1024 ZiB

This new naming and definition aim to eliminate the ambiguity when using traditional storage units, although “KB”, “MB”, and “GB” are still widely used to represent storage sizes calculated both by powers of 2 and powers of 10.

1 KiB (Kilobyte) 2^10 = 1024 Bytes

When the number reaches 1024, we enter the world of ‘KiB’. This is not just a quantitative leap but also marks a new dimension in data storage.

1 MiB (Megabyte) 2^20 = 1024 KiB

Doubling up, we quickly jump from KiB to MiB. For those who grew up in the era of floppy disks, a 1.44MB floppy disk was once the way to bring treasure to school.

1 GiB (Gigabyte) 2^30 = 1024 MiB

Then comes GiB. To watch a high-definition movie, you might need several GiB of storage space. It’s hard to imagine that in the near future, ‘TiB’ (Terabyte) will become commonplace.

Others

Octal and Hexadecimal

In the traditional mindset of programmers, octal (base 8) and hexadecimal (base 16) are often used to simplify the representation of binary code. Hexadecimal is particularly popular because it allows a four-bit binary number to be represented as a single character, making binary code easier to read and write. Octal uses 8 digits (0 to 7) to represent, while hexadecimal uses 16 digits (0 to 9 and A to F).

Hexadecimal is often used when dealing with memory addresses or color values. For example, the binary 1111 1111 can be represented as FF in hexadecimal. In web development, color values are often represented in hexadecimal, such as #FF5733, representing a specific color.

January 1, 1970: The Starting Point of Epoch Time

In computer systems, time is usually represented with January 1, 1970, as the starting point, known as the Unix epoch. The number of seconds since that day is used to represent time and date. 32-bit systems use 32 bits to store timestamps (the number of seconds since January 1, 1970), meaning they can represent the maximum timestamp of (2^31 - 1) seconds, because one bit is used to indicate sign. This will cause a “Y2K” style problem on January 19, 2038, when the system time will overflow, potentially causing software and system errors.

Y2K Bug

As the year 2000 approached, the ‘Y2K bug’ became a hot topic worldwide. The root of the problem was that many early computer systems used only two digits to represent years, leading to concerns about global data chaos. In hindsight, these worries may have been overblown, but they did highlight the vulnerability of relying on digital technology.

Quantum Computing

Lastly, let’s venture into a completely new realm—quantum computing. In the quantum computing model, ‘qubits’ can exist in both 0 and 1 states simultaneously, breaking the traditional framework of computing and offering new possibilities for solving certain types of problems.

Conclusion

The virtual world built from 1s and 0s is more vast and complex than our daily lives. Exploring the world of these magical numbers is not just about satisfying curiosity; it’s about discovering and creating new technologies and solving real-world problems. The story of numbers in the field of computing continues, and we are standing at the starting point of one exciting journey after another.
